## Security Reviewer-gpt
**Status**: changes-requested  
**Phase**: security-review  
**Findings**:
- **medium** — `routers/api.py:600-606` and `services/scoring.py:44-46`: `operator` is checked with `operator not in valid_operators` without first validating it is a string. If a request or persisted JSON supplies an unhashable value such as `[]` or `{}`, Python raises `TypeError`, causing a 500 response/crash instead of a controlled validation failure or sanitized skip.  
  **Remediation**: require `isinstance(operator, str)` before set membership. In the route, reject non-string operators with HTTP 422. In persisted-threshold sanitization, skip non-string operators.
- **info** — Deleted parser helpers / `get_universal_criteria()` rename: no remaining references to `suggest_criteria`, `extract_thresholds`, or `extract_thresholds_detailed` found.
- **info** — Added config/dependencies/scripts: `ruff==0.14.9` is pinned in `requirements-dev.txt`; `frontend/package.json` adds only `typecheck: "tsc --noEmit"`. No overly broad or obviously unsafe scripts identified.
- **info** — No hardcoded secrets or newly exposed sensitive files identified in reviewed changes. Existing `optium.db` is covered by `.gitignore`.

**Approved**: no
